## 1. Thesis (one sentence)

A direct multi-horizon CatBoost forecaster that injects the *forecasted weather at each target
hour* and unifies the train/serve weather source (Open-Meteo) matches/beats ISO-New England's
operational day-ahead forecast for the NEMA zone, once the comparison is made horizon-matched.

## 2. Contributions (what is novel / worth publishing)

1. Direct per-horizon design that removes recursive error compounding (80.6% lower avg MAE
   than a single-model roll-out across 24 h).
2. Target-hour exogenous (calendar + forecast-weather) features as the lever for day-ahead
   accuracy (day-ahead MAE 183 → 77 MW).
3. Identification and correction of a **train/serve weather-source mismatch** (live day-ahead
   147 → 90 MW) — a deployment lesson rarely reported.
4. A **horizon-matched** evaluation protocol vs an operational ISO forecast, plus a rigorous
   leakage audit (shuffled-target test).
5. A fully reproducible, keyless, deployable pipeline (ISO-NE Web Services + Open-Meteo).

## 5. Writing workflow (how the agents are used)

1. **Scaffold (this commit):** `FACTS.md`, `PLAN.md`, `main.tex`, `references.bib`,
   `make_figures.py`, generated figures, empty `sections/`.
2. **Drafting (agents):** spin up parallel general-purpose agents, each owning a contiguous
   block of sections. Every agent (a) reads `paper/FACTS.md`, (b) writes only LaTeX into its
   assigned `paper/sections/*.tex` files, (c) uses ONLY numbers from FACTS.md, (d) adds
   `\label{}` for cross-refs, (e) academic register, no first-person plural overuse.
   - Agent A → `00_abstract`, `01_introduction`, `02_related_work`
   - Agent B → `03_data`, `04_methodology`
   - Agent C → `05_experimental_setup`, `06_results`
   - Agent D → `07_discussion`, `08_limitations`, `09_conclusion`
3. **Assembly:** `main.tex` already `\input`s the section files; fill `references.bib`.
4. **Review pass:** verify every number matches FACTS.md; fix cross-refs; ensure it compiles
   on Overleaf (`pdflatex`); tighten length.
5. **Commit & push.**
